     1→# Cover Design - Cover Layout Engine Specification
     2→
     3→> The cover is the first impression. Either skip it, or build it like architecture.
     4→
     5→---
     6→
     7→## ⚠️ Critical Rules (Read First)
     8→
     9→1. **Cover is OPTIONAL.** Do NOT force a cover on documents that don't need one. When in doubt, skip.
    10→2. **Unified cover system.** All routes (Report, Creative, Academic) use the HTML/Playwright cover system. **Template 01** is general-purpose (also covers government/bidding scenarios); **Template 07 is Crystal Blue (dark cover + light-blue body)**; **Templates 03-04** are academic-specific (dark backgrounds, scholarly typography; 04 also covers Chinese journal/keyword scenarios); **Template 06 is institutional (white bg, black border frame, structured fields)**. All routes generate covers via Playwright and merge via pypdf.
    11→3. **Single PDF output.** Never deliver a separate cover PDF.
    12→   - **Creative route**: Cover is part of the same HTML document → single PDF output inherently.
    13→   - **Report route**: Cover PDF (Playwright) + body PDF (ReportLab) → merged via pypdf into one final PDF.
    14→   - **Academic route**: Cover PDF (Playwright) + body PDF (Tectonic) → merged via pypdf into one final PDF.
    15→4. **Page isolation.** Cover content must NEVER share a page with TOC, body text, or any subsequent content. Report/Academic: isolation is inherent in the merge pipeline. Creative: CSS page-break enforces isolation. Cover + TOC on the same page = **critical bug**.

    33→# PART 0: GLOBAL ENGINE ARCHITECTURE (Mandatory)
    34→
    35→These three architectural upgrades are **mandatory** for ALL cover rendering. They eliminate 90% of squished/misaligned/overflow bugs at the engine level.
    36→
    37→---
    38→
    39→## A0.0 - Global Base Parameters
    40→
    41→**All templates MUST obey these.**
    42→
    43→```
    44→W = page total width
    45→H = page total height
    46→U = W * 0.05           # Base spacing unit (5% of page width)
    47→                        # All spacing should be multiples of U
    48→```
    49→
    50→**Coordinate origin:** `(0, 0)` at top-left corner. Some PDF libraries (ReportLab) use bottom-left origin - invert Y axis accordingly: `y_pdf = H - y_spec`.
    51→
    52→---
    53→
    54→## A0.1 - Absolute Anchor Grid (Replaces Flow Layout)
    55→
    56→**Old (DEPRECATED):** `Y = previous_element_Y + height + spacing` - if one element overflows, everything below shifts and gets crushed.
    57→
    58→**New (MANDATORY):** Page height = 100%. Every element group gets an **absolute percentage Y-anchor**. Elements may only grow **within their own bounding box**. They NEVER push or compress other blocks.
    59→
    60→### Implementation Rule
    61→
    62→```
    63→# WRONG - flow layout (banned)
    64→y_cursor = PAGE_H - top_margin
    65→y_cursor -= title_height
    66→y_cursor -= gap
    67→y_cursor -= subtitle_height   # ← if title wraps, subtitle gets crushed
    68→
    69→# RIGHT - absolute anchor grid
    70→ANCHOR_TITLE_Y  = H * 0.30
    71→ANCHOR_SUMMARY_Y = H * 0.50
    72→ANCHOR_META_Y   = H * 0.70
    73→ANCHOR_FOOTER_Y = H * 0.90
    74→# Each block renders at its anchor, independent of others
    75→```
    76→
    77→### Bounding Box Containment
    78→
    79→Each block has a **maximum bounding box** defined by two consecutive anchors:
    80→- Block can grow downward within `[own_anchor_Y, next_anchor_Y - min_gap]`
    81→- If content exceeds its bounding box → trigger overflow protection (see Part 3)
    82→- Blocks NEVER consume space belonging to adjacent blocks
    83→

    85→
    86→## A0.2 - Typography Weight & Spacing System
    87→
    88→Use **weight**, **letter-spacing**, and **opacity** to create hierarchy - not just font size.
    89→
    90→### Mandatory Type Roles
    91→
    92→| Role | Size | Weight | Letter-Spacing | Line-Height | Opacity | Purpose |
    93→|------|------|--------|----------------|-------------|---------|---------|
    94→| **Kicker / Footer** (decorative text) | 16pt | Regular | 3pt (very wide) | - | 60% | Wide spacing + transparency makes 16pt text feel delicate and recessive |
    95→| **Summary / Description** (summary paragraph)  | 16-18pt | Regular | normal | **1.6** | 85% | **Fill visual space** - 2-4 lines of descriptive text that prevents empty covers |
    96→| **Meta / Subtitle** (secondary text) | 20-22pt | Light / Regular | normal | 1.4 | 85% | Comfortable reading rhythm, clear secondary hierarchy |
    97→| **Hero Title** (main title) | 32-65pt (CJK: 32-80pt) | Black / Heavy (extra bold) | normal-tight | **1.15** (multi-line) | 100% | Must create overwhelming scale contrast; visually dominates the page. CJK characters need +15-20% size to match Latin visual weight. **⚠️ For 3-line CJK titles with subtitle, max ~42pt (see S3.1 table).** |
    98→
    99→###  Data-to-Drawer Binding Rule
   100→
   101→> **Hero Title = Company/entity name. Kicker = Report type/subtitle. Never reverse.**
   102→
   103→When users provide structured information (company name + report name/type), they must be bound to typography drawers by these rules:
   104→
   105→| User Data Field | Bound to Typography Role | Notes |
   106→|-------------|-------------|------|
   107→| **Company/entity name** (e.g. "GREENTECH") | **Hero Title** (45-65pt Heavy) | Company name is the **absolute visual center**, largest font, heaviest weight |
   108→| **Report type/subtitle** (e.g. "2025 Annual Report Summary") | **Kicker** (16pt, letter-spacing 3pt, opacity 60%) | Report name is decorative supplementary text, placed in small text position at top-left/above title |
   109→| **Summary/description** | **Summary** (16-18pt) | Detailed description text |
   110→| **Date/author/version** | **Meta** (20-22pt) | Auxiliary information |
   111→| **Document number/org signature** | **Footer** (16pt, opacity 60%) | Bottom closing |
   112→
   113→**Mapping priority (when ambiguous):**
   114→1. If user provides only one name → treat as company/entity name, bind to Hero Title
   115→2. If user provides two names → shorter/more brand-like → Hero Title; longer/descriptive → Kicker or Summary
   116→3. If user explicitly labels "title" and "subtitle" → title → Hero Title, subtitle → Kicker
   117→4. **Never use report type names (e.g. "Annual Report", "White Paper") as Hero Title's largest text** - report type is always Kicker-level decorative text
   118→
   119→### The Summary Block Rule (Anti-Void Iron Rule) 
   120→
   121→> Every cover MUST include a Summary/Description text block. If the user provides no summary, the system MUST auto-generate one.
   122→
   123→**Why:** A cover with only a title and date looks barren. The Summary block physically fills 2-4 lines of space, preventing the "empty field" aesthetic.
   124→
   125→**Auto-generation rule:** When no summary/description is provided:
   126→```python
   127→# Generate a default summary
   128→if not summary_text:
   129→    summary_text = f"This report was generated by {org_name or 'the system'}, containing comprehensive data analysis and insights."
   130→    # Or in English:
   131→    # summary_text = f"This report presents comprehensive analysis and key insights prepared by {org_name or 'the organization'}."
   132→```
   133→
   134→**Constraints:**
   135→- Width: template-specific (typically `W * 0.5` to `W * 0.6`)
   136→- Lines: 2-4 lines (auto-wrap at width boundary)
   137→- Never truncate summary - if too long, reduce to 4 lines max with `...`
   138→
   139→### Font Weight Fallback
   140→
   141→- If font family lacks Black/Heavy weight → use Bold + slightly larger size (+4pt)
   142→- If font family lacks Light weight → use Regular + increased letter-spacing (+1pt)
   143→- CJK fonts: Noto Serif SC for Regular/Light roles; for Heavy/Black → increase size by 15% to compensate for single-weight CJK fonts
   144→- English kickers/footers: **FORCE UPPERCASE** via code (`text.upper()` / `toUpperCase()`)

   148→## A0.3 - Z-Index Layer Management
   149→
   150→All cover elements must be rendered in strict layer order. No exceptions.
   151→
   152→| Layer | Z-Index | Contents | Rules |
   153→|-------|---------|----------|-------|
   154→| **Layer 0** (base) | 0 | Background fill (white / light gray) | Always rendered first; full page |
   155→| **Layer 1** (background) | 1 | Grids, watermark letters, decorative blocks, large clipped graphics | **MUST enable clip-path** - background elements may extend beyond logical bounds but must be clipped to page physical bounds. Never let background elements inflate PDF page size. |
   156→| **Layer 2** (structure) | 2 | Ultra-thin divider lines, sidebars, corner crop marks | Structural guides that define spatial zones |
   157→| **Layer 3** (content) | 3 | All readable text content | Rendered last, always on top |
   158→
   159→### Clip-Path Enforcement
   160→
   161→```css
   162→/* clip background overflow — ONLY on Layer 1 */
   163→.cover-bg-layer {
   164→  position: absolute;
   165→  inset: 0;
   166→  overflow: hidden;  /* MANDATORY */
   167→  z-index: 1;
   168→}
   169→```
   170→
   171→> ⚠️ **Clip scope = Layer 1 ONLY.** `overflow: hidden` must ONLY be on the Layer 1 background container. Layer 2 (lines) and Layer 3 (text) containers must NOT have `overflow: hidden`.
   172→
   173→###  Anti-Clip Bug (Layer 3 Text Truncation Fix)
   174→
   175→**Symptom:** Cover text is visible but truncated by an invisible boundary.
   176→
   177→**Root cause:** `overflow: hidden` scope too broad, clipping text in Layer 3.
   178→
   179→**Iron rule:**
   180→```html
   181→<!-- ✅ CORRECT - overflow:hidden only on Layer 1 -->
   182→<div class="cover-layer-1" style="position:absolute; inset:0; overflow:hidden; z-index:1;">
   183→  <!-- Background decorative elements -->
   184→</div>
   185→<div class="cover-layer-2" style="position:absolute; inset:0; z-index:2;">
   186→  <!-- Structure lines, no overflow:hidden -->
   187→</div>
   188→<div class="cover-layer-3" style="position:absolute; inset:0; z-index:3;">
   189→  <!-- Text content, no overflow:hidden -->
   190→</div>
   191→
   192→<!-- ❌ WRONG - global overflow:hidden clips text -->
   193→<div class="cover-container" style="overflow:hidden;">
   194→  <div class="layer-1">...</div>
   195→  <div class="layer-2">...</div>
   196→  <div class="layer-3">...</div>  <!-- Text gets clipped! -->
   197→</div>
   198→```
   199→
   200→###  No Page Border/Frame
   201→
   202→**Symptom:** A rectangular border appears around the entire cover page.
   203→
   204→**Iron rule:**
   205→```css
   206→.cover-page {
   207→  border: none !important;
   208→  outline: none !important;
   209→  box-shadow: none !important;
   210→}
   211→```
   212→
   213→###  Minimum Spacing Between Decorative Lines and Text (Line-to-Text Spacing)
   214→
   215→**Symptom:** Decorative lines on the cover (Layer 2 dividers, corner marks, sidebar edges) are flush against or overlapping with text.
   216→
   217→**Iron rule:**
   218→```
   219→Minimum spacing between decorative lines and any text content = U (= W * 0.05)
   220→i.e., at least 1 U of whitespace between line edges and text edges
   221→```
   222→
   223→| Line Type | Minimum Spacing | Notes |
   224→|---------|---------|------|
   225→| Horizontal divider | `U` above and below the line | Line must not be flush against title or body text |
   226→| Vertical sidebar edge | `U` to the right of the line | Text inside sidebar must maintain spacing from the edge |
   227→| Ultra-thick anchor line (Template 01) | `2 * U` to the right of the line | Thick line and title need ample breathing room |

   241→## Template 01: HUD Data Terminal - Ultra-Thick Vertical Anchor Line
   242→
   243→**Design intent:** A single bold vertical line on the left anchors all visual weight. The thick line eliminates left-side floating. Clean, data-driven, authoritative.
   244→
   245→### Layer 1 - Background
   246→- Full-page grid pattern: horizontal + vertical lines at ~50pt intervals
   247→- Grid color: primary color at **2% opacity** (white background, nearly invisible)
   248→- Grid line width: `0.5pt`
   249→
   250→### Layer 2 - Structure
   251→- **Left anchor line:** Start `(0.12*W, 0.1*H)`, End `(0.12*W, 0.9*H)`. Line width = **6pt**, primary color.
   252→
   253→
   254→### Layer 3 - Content
   255→
   256→**Content left edge: `X_content = 0.12*W + 30pt`** (offset from the thick line)
   257→
   258→| Drawer | Y-Anchor | Content | Constraints |
   259→|--------|----------|---------|-------------|
   260→| **A - Kicker** | `0.15 * H` | Report type / subtitle (e.g. "2025 Annual Report Summary") | 16pt, Regular, letter-spacing 3pt, opacity 60%, uppercase |
   261→| **B - Hero Title** | `0.30 * H` | **Company/entity name** (e.g. "GREENTECH") | 45-65pt (CJK: 50-80pt), Heavy. Company name is the visual center |
   262→| **C - Summary**  | `0.50 * H` | 2-3 lines descriptive text about the report | 16-18pt, Regular, line-height 1.6, opacity 85%. **Width limit: `W * 0.6`**, auto-wrap. This drawer fills the mid-page void |
   263→| **D - Meta/Date** | `0.75 * H` | Author, org, date | 16-20pt, Regular |
   264→
   265→### Best For
   266→Technology reports, data analysis, dashboard summaries, technical white papers

   295→# PART 3: CODE-LEVEL SAFETY MEASURES (Pre-Render Safeguards)
   296→
   297→These checks run BEFORE final rendering. They are **mandatory** - not optional optimizations.
   298→
   299→---
   300→
   301→## S3.0 - Cover Overlap Validation (MANDATORY)
   302→
   303→**After generating the cover HTML and before rendering to PDF, run:**
   304→
   305→```bash
   306→node "$PDF_SKILL_DIR/scripts/cover_validate.js" cover.html
   307→```
   308→
   309→This detects text-vs-decorative-line overlaps by rendering the HTML and measuring actual bounding boxes. Exit code 1 = overlap found, must fix before generating the PDF.
   310→
   311→The minimum gap between any text element and any decorative line is **1U (= 5% of page width ≈ 40px on A4)**. This catches the exact bug shown in the "text overlapping decorative lines" screenshots.
   312→
   313→**If the check fails:**
   314→1. Adjust the decorative line's Y position to maintain ≥ 1U gap from the nearest text
   315→2. Or adjust the text block's position/size to avoid the overlap
   316→3. Re-run `cover_validate.js` until it passes (exit code 0)
   317→
   318→---
   319→
   320→## S3.1 - Hero Title Overflow Protection (Title Line-Wrapping)
   321→
   322→**Rule:** Hero title must NEVER exceed its template's width boundary.
   323→
   324→1. If rendered width > max_width → word-wrap (CJK: any character; Latin: space/hyphen)
   325→2. Multi-line hero titles: **lock line-height to 1.15**
   326→3. Maximum lines: **3** — if 4+ lines needed, reduce font size by 4pt increments
   327→4. Minimum font size floor: **32pt** (below this, truncate with `...`)
   328→
   329→### Title Size vs Lines Quick-Reference (Template 01, zone B→C ≈ 225px)
   330→
   331→| Lines | With subtitle | Without subtitle |
   332→|-------|--------------|------------------|
   333→| 1 | ≤ 80pt | ≤ 80pt |
   334→| 2 | ≤ 55pt | ≤ 70pt |
   335→| 3 | ≤ 42pt | ≤ 48pt |
   336→
   337→**⚠️ CJK 3-line titles must reduce to ~42pt.** Do NOT use 50-80pt for 3-line CJK titles — it WILL overlap the Summary zone.
   338→
   339→---
   340→
   341→## S3.2 - Zone Collision Detection
   342→
   343→After placing each text block, check if its bottom edge penetrates the next zone's top:
   344→
   345→1. **Reposition first:** Push the collided zone down by `overlap + 20px` gap. All downstream zones shift together.
   346→2. **Font reduction (if reposition insufficient):** Decrease title by 2pt increments. Floor: **32pt** for titles, 14pt for meta/summary.
   347→3. **Truncation (last resort):** If font reduction hits floor and still collides, truncate with `...`
   348→
   349→---
   350→
   351→## S3.3 - Uppercase Lock
   352→
   353→**Force-uppercase when content is English/Latin:**
   354→- Kicker (category label / lead-in text)
   355→- Footer (closing date / document number)
   356→- Background watermark text
   357→- Any Layer 1 decorative text
   358→
   359→CJK text is exempt. Mixed CJK+Latin strings: uppercase only the Latin portions. Use `text-transform: uppercase` in CSS or `.upper()` in code.

   377→## S3.6 - Background Watermark Full-Display Enforcement 
   378→
   379→**All watermark text in Layer 1 must be 100% within the visible page area. Cropping is strictly forbidden.**
   380→
   381→**Applicable scope:**
   382→- `cover-backgrounds.md` Recipe 2.1 giant sidebar pillar
   383→- `cover-backgrounds.md` Recipe 2.2 bottom full-size text
   384→- Any other decorative text in the background layer

   387→1. Horizontal text: rendered width must not exceed `W * 0.90` (5% safety margin each side)
   388→2. Vertical/rotated text: rendered height must not exceed `H * 0.85` (7.5% safety margin top/bottom)
   389→3. If exceeded, scale down font size proportionally — never truncate
   390→4. Anchor coordinates must never exceed page boundaries

   394→## S3.7 - Line-Length Alignment (Line Must Match Text Span)
   395→
   396→**Decorative lines must be sized relative to the text they serve:**
   397→
   398→- **Vertical lines:** height = text block height (± 1U padding above/below first/last element)
   399→- **Horizontal lines:** width ≥ widest text element in its zone (up to 120%, NEVER shorter)
   400→
   401→```css
   402→.vline {
   403→  position: absolute;
   404→  top: var(--content-top);     /* align with first text element */
   405→  bottom: var(--content-bottom); /* align with last text element */
   406→}
   407→.hline {
   408→  width: max(100%, 200px);     /* at least as wide as parent text container */
   409→}
   410→```
   411→
   412→**Checklist:**
   413→- [ ] Every vertical line’s height matches its adjacent text block span
   414→- [ ] Every horizontal line’s width ≥ widest text element in its zone
   415→- [ ] No decorative line is shorter than the text it accompanies
   416→

   419→## S3.8 - Vertical Balance (Anti-Top-Heavy Layout)
   420→
   421→**Problem:** Content clusters at top of page, bottom 40%+ is dead whitespace.
   422→
   423→**Rule:** When total content height < 50% of page height, switch to centered distribution (flexbox `justify-content: center`). Otherwise use anchor grid:
   424→
   425→| Content Volume | Title Anchor | Summary Anchor | Meta Anchor |
   426→|---------------|-------------|----------------|-------------|
   427→| **Sparse** (fill < 50%) | Centered mode | Centered mode | Centered mode |
   428→| **Normal** (fill 50-80%) | `H * 0.30` | `H * 0.48` | `H * 0.70` |
   429→| **Dense** (fill > 80%) | `H * 0.20` | `H * 0.40` | `H * 0.65` |
   430→
   431→**CJK Title Size Compensation:** CJK characters appear smaller at same pt size. Increase by 15-20%:
   432→- CJK Hero Title: 50-80pt (Latin: 45-65pt)
   433→- CJK Kicker: 11-12pt (Latin: 9pt)
   434→- CJK Summary: 17-20pt (Latin: 16-18pt)
   435→
   436→**Checklist:**
   437→- [ ] No cover has >40% dead whitespace at the bottom
   438→- [ ] CJK titles are 15-20% larger than equivalent Latin titles
   439→- [ ] Sparse-content covers use centered distribution
   440→
   441→---
   442→
   443→## S3.9 - Percentage Positioning Requires Known-Size Container
   444→
   445→**Problem:** `top: XX%` resolves against containing block’s height. If height is undefined (all children absolutely positioned), percentages collapse and elements stack.
   446→
   447→**Iron rule:** Container with `top: XX%` children MUST have deterministic height (`height: 100%`, `inset: 0`, or `top` + `bottom` pair).
   448→
   449→```css
   450→/* ✅ Safest: flat structure with px values */
   451→.cover { position: relative; width: 794px; height: 1123px; }
   452→.kicker { position: absolute; top: 225px; left: 95px; }
   453→.title  { position: absolute; top: 292px; left: 95px; }
   454→
   455→/* ✅ OK: wrapper with inset:0 gives it known height */
   456→.content-left { position: absolute; inset: 0; width: 55%; }
   457→.title { position: absolute; top: 26%; }  /* 26% of 1123px ✓ */
   458→
   459→/* ❌ BANNED: wrapper without height, percentage is undefined */
   460→.content-left { position: absolute; left: 12%; top: 0; width: 55%; }
   461→.title { position: absolute; top: 26%; }  /* 26% of WHAT? → overlap */
   462→```

   466→# PART 4: COVER COLOR RULES
   467→
   468→> Cover colors must be consistent with the body color system - they cannot exist independently.
   469→> **The cascade palette CSS output includes `--c-*` aliases** that map directly to cover template variables. Paste the CSS output into cover HTML's `:root` block — template 01 works without any manual color translation.
   470→
   471→```
   472→Cover primary    = Body theme color
   473→Cover secondary  = Primary lightness variant (±20% lightness)
   474→Cover background = Pure white / very light gray / primary at 5-8% opacity
   475→```
   476→
   477→### Absolutely Forbidden
   478→
   479→- ❌ Dark large-area solid backgrounds (dark blue, dark green, black filling the page)
   480→- ❌ Gradient backgrounds (any `linear-gradient` / `radial-gradient` as large-area fill)
   481→- ❌ High-saturation color schemes
   482→- ❌ Rainbow / multi-color gradients

   881→2. **Institution name**: Centered, serif, 28-34pt Bold, letter-spacing 4-6px. For CJK names, use wider letter-spacing (6px). For Latin names, use standard (3px).
   882→3. **Thick divider**: 2pt solid line, 70% width, separates institution from content below.
   883→4. **Document type**: 20-24pt, lighter weight than institution name, letter-spacing 3-4px. This slot differentiates document categories (e.g. "开题报告" / "Thesis Proposal" / "毕业论文" / "Graduation Design").
   884→5. **Title**: Serif, 26-30pt Bold, max 3 lines, centered. Auto-wrap at 520px width.
   885→6. **Structured fields**: Left-aligned label + centered underline value. Label width fixed by longest label in the set. 3-7 field rows supported. Common fields: Name/姓名, Student ID/学号, Advisor/导师, Department/院系, Major/专业.
   886→7. **Date**: Centered at bottom, 14pt, letter-spacing 2px.
   887→
   888→**Field auto-detection:**
   889→When the user provides structured metadata (name, student ID, advisor, etc.), auto-populate the fields block. When no fields are provided, omit the fields-block entirely and let the title expand vertically into the freed space.
   890→
   891→**Variant 06B - Double border:**
   892→For extra formality (government documents, official submissions), replace the single border with a double border (outer 2.5pt + inner 1pt, 6px gap):
   893→```css
   894→.border-frame {
   895→  border: 2.5px solid var(--c-accent);
   896→  outline: 1px solid var(--c-accent);
   897→  outline-offset: 6px;
   898→}
   899→```
   900→
   901→**This template has NO background decoration layer, NO watermarks, NO gradients.** The black frame + white space IS the design.
